# ChanGPT CMS & hosting setup

The site is hosted on **GitHub Pages**, built & deployed automatically by **GitHub
Actions** on every push, with a content editor (**Decap CMS**) at **`/admin`**.
Login uses a small **Cloudflare Worker** (`cms-auth-worker/`). No server to run.

```
Edit at /admin ─► commit to GitHub ─► Actions builds & deploys ─► GitHub Pages ─► Cloudflare DNS ─► visitors
                       login handshake ▲
                       Cloudflare Worker (cms-auth.changpt.org)
```

## One-time setup

### 1. Turn on GitHub Pages (Actions source)
Repo → **Settings → Pages → Build and deployment → Source: GitHub Actions**.
Push to `main` (or run the workflow manually) — the `Deploy site to GitHub Pages`
workflow builds and publishes `build/`.

### 2. Point the domain at GitHub Pages (Cloudflare DNS)
- Repo → **Settings → Pages → Custom domain** → `changpt.org` (the `static/CNAME`
  file already sets this too).
- In **Cloudflare DNS**, point `changpt.org` at GitHub Pages — easiest is a
  `CNAME changpt.org → chiweic.github.io` (Cloudflare flattens CNAMEs at the apex).
- Set that record to **DNS only (grey cloud)** so GitHub can issue the HTTPS cert;
  then enable **Enforce HTTPS** in the Pages settings. (You can re-enable
  Cloudflare proxy later with SSL mode = Full.)
- `app.changpt.org` is untouched.

### 3. Create a GitHub OAuth App (for the CMS login)
GitHub → **Settings → Developer settings → OAuth Apps → New OAuth App**
- Homepage URL: `https://changpt.org`
- Authorization callback URL: `https://cms-auth.changpt.org/callback`

Note the **Client ID** and generate a **Client Secret**.

### 4. Deploy the login Worker
```bash
cd cms-auth-worker
npm install
npx wrangler login
npx wrangler secret put GITHUB_CLIENT_ID       # paste Client ID
npx wrangler secret put GITHUB_CLIENT_SECRET   # paste Client Secret
npx wrangler deploy
```
Then in the Cloudflare dashboard: **Workers & Pages → changpt-cms-auth →
Settings → Domains & Routes → Add → `cms-auth.changpt.org`** (auto-creates DNS).
This must match `base_url` in `static/admin/config.yml`.

### 5. Edit content
Go to **`https://changpt.org/admin`**, log in with GitHub. Publishing commits
Markdown to the repo → Actions rebuilds → live in ~1–2 min.

## Notes
- **Access control:** only GitHub users with write access to `chiweic/changpt-site`
  can save edits. No separate user database.
- **Repo visibility:** public repos get unlimited Actions minutes + free Pages.
  Private repos get a monthly free allowance.
- Images uploaded in the editor are committed to `static/img/uploads/`.
- New docs created in `/admin` appear in the sidebar automatically (it's
  autogenerated, ordered by each doc's "sidebar position").
